# Brainstorming for Meaningful Decisions

Use lightweight design only when it prevents real rework.

## Use this skill for

- New features with multiple reasonable approaches.
- Experiment design, architecture, data model, permissions, or irreversible decisions.
- User-facing behavior changes where the goal or success criteria are unclear.

## Do not use this skill for

- Questions, status checks, translations, or explanations.
- Small documentation, configuration, or one-file reversible edits.
- Installing, copying, or updating an already selected Skill.
- Work that follows a user-approved plan with no material decision left.

## Workflow

1. Read only the relevant project rules and current files.
2. Ask one focused question only if an unanswered decision would materially change the result.
3. State the recommended approach, key trade-off, and success check in a few lines.
4. For costly, risky, or direction-changing work, wait for approval before implementation. For ordinary reversible work the user has already asked to perform, proceed.

Do not require a design document, visual companion, or multi-round review unless the user asks or the task genuinely needs it.
