# 安全查找 Skills

`find-skills-safe` 用于寻找适合某项任务的 Skill、插件或 MCP，并在采用前做安全审查。

## 它会做什么

1. 明确你的具体任务、使用平台和关键限制。
2. 优先核查官方文档、官方市场和原作者仓库。
3. 阅读候选的 `SKILL.md`、脚本和配置，检查联网、权限、凭据、数据访问、依赖和许可证。
4. 一次只给出一个候选，并说明用途、证据、风险和是否值得采用。

## 它不会做什么

- 不执行外部命令或第三方脚本。
- 不调用包管理器。
- 不安装、更新、删除 Skill 或插件。
- 不改全局设置，不绕过确认。

“查找”和“安装”是两个阶段。只有你在之后明确要求安装某一个候选时，才会进入独立的安装流程，并重新检查来源、权限和写入位置。

## 示例

- “找一个能审查 GitHub PR 的 Skill，只推荐一个并检查权限。”
- “找一个适合 Codex 的 Figma 插件，告诉我它会访问什么数据。”
- “找一个可连接本地文档的 MCP；不要把文件上传给第三方。”

安装量、榜单排名、星标或安全标签只能作为参考信号，不是安全证明。来源无法核验、包含无法审查脚本、权限过宽或风险无法解释的候选，应标记为不推荐或证据不足。
