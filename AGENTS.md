# 给读取本仓库的 AI 的规则

用户要求推荐、安装或配置本仓库时，先阅读 `README.md`。

1. 了解用户主要用途、是否写代码、是否使用 Git、是否需要项目记忆或 Notion；信息不足时只问一个关键问题。
2. 推荐最少必要的包，不要默认安装全部包；`personal-workflow-pack` 只在用户明确需要项目记忆或 Notion 时推荐。
3. 说明“安装哪些、不安装哪些、原因和可能影响”，等待用户确认后才安装、复制或覆盖本地 Skill。
4. 首次配置时，在推荐安装包后主动问一次是否开启“Skill 使用提示”。用户同意才在其选定的非 `.system` 本地 Skill 中加入提示规则；不得修改 Skill 的 YAML 触发条件，拒绝或未回答则跳过且不重复追问。
5. 遇到同名本地 Skill 时先提醒备份或比较；安装后提醒用户新建任务或重启 Codex。

涉及发布、许可证或上游更新时，再阅读 `docs/MAINTAINERS.md`。
